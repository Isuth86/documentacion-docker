# Documentación de Servicios con Docker

Bienvenido a la documentación técnica del proyecto de Administración de Sistemas Operativos.

## Objetivo del Proyecto

Este proyecto tiene como objetivo documentar el despliegue y configuración de servicios esenciales utilizando **Docker**. Específicamente, se cubren dos casos prácticos:

1.  **LAMP + WordPress**: Creación de un entorno de servidor web completo (Linux, Apache, MySQL/MariaDB, PHP) para alojar un sitio de WordPress en un único contenedor.
2.  **WordPress + MariaDB (Contenedores Separados)**: Despliegue modular utilizando una red Docker para conectar un contenedor de aplicación y otro de base de datos.
3.  **Portainer CE**: Implementación de una interfaz gráfica para gestionar contenedores Docker de manera eficiente.

## Estructura de la Documentación

La documentación está organizada en las siguientes secciones:

*   **Introducción**: Visión general de las tecnologías utilizadas.
*   **Instalación y Requisitos**: Pasos detallados para instalar y configurar los contenedores y servicios.
*   **Uso y Funcionamiento**: Guía sobre cómo acceder y utilizar los servicios desplegados.
*   **Conclusiones**: Resumen del aprendizaje y posibles mejoras.

!!! info "Nota importante"
    Esta documentación está basada en las prácticas realizadas por **Ismael Bermúdez Del Río** para el módulo de Administración de Sistemas Operativos.

## Tecnologías Clave

| Tecnología | Descripción |
| :--- | :--- |
| **Docker** | Plataforma de contenerización. |
| **Apache** | Servidor web HTTP. |
| **MariaDB** | Sistema de gestión de bases de datos. |
| **WordPress** | Sistema de gestión de contenidos (CMS). |
| **Portainer** | Gestor de contenedores con interfaz web. |
